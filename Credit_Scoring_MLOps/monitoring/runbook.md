# Руководство по реагированию на инциденты в сервисе кредитного скоринга

Этот документ объясняет последовательность действий при активации оповещений Prometheus для сервиса `credit-api`.

---

## 1. Высокий уровень ошибок в CreditApi

**Оповещение:** `CreditApiHighErrorRate`  
**Критерий:** Процент ответов с кодом 5xx превышает 5% в течение последних 5 минут.  
**Уровень критичности:** `warning`

### Этапы диагностики

1. Перейти в Grafana и открыть панель управления `Credit API / Overview`.
2. Изучить следующие элементы:
   - график интенсивности входящих запросов,
   - график распределения кодов ответов (2xx / 4xx / 5xx),
   - временной момент, когда начался рост ошибок 5xx.
3. Просмотреть логи приложения (стандартный вывод контейнера `credit-api`):
   ```bash
   docker logs credit-api --since=10m
